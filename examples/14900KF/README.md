# VectorFFT calibrated wisdom — Intel i9-14900KF (Raptor Lake, AVX2)

## What's in this folder

`wisdom.txt` — 198 calibrated plans for the (N, K) grid covered by the
v1.0 benchmark suite. Generated by running `python build_tuned/calibrate.py`
on a single P-core of an Intel Core i9-14900KF (Raptor Lake, AVX2,
DDR5-6400, performance power plan, no concurrent load).

Loading this file lets `VFFT_MEASURE` plans hit the wisdom cache and
skip per-cell calibration on first run — instead of "minutes-to-hours
on first plan creation" you get "sub-millisecond plan time, headline
performance". Without it, `VFFT_MEASURE` calibrates each cell on first
use (and persists in the in-memory database).

## How to use

```c
#include <vfft.h>

vfft_init();
if (vfft_load_wisdom("examples/14900KF/wisdom.txt") == 0) {
    /* Wisdom loaded — plans below hit the cache, sub-ms creation. */
} else {
    /* File missing or unreadable — VFFT_MEASURE will calibrate
       per cell on first use. Or fall back to VFFT_ESTIMATE. */
}

vfft_plan p = vfft_plan_c2c(1024, 256, VFFT_MEASURE);
/* ... */
```

The library does **not** auto-load any wisdom file. Users must opt in
explicitly. The path above is relative to the working directory — if
your binary runs from somewhere else, pass an absolute path or wire it
through your build (the `examples/quickstart.c` demo bakes the path at
compile time via a CMake `-D` define for an example of this pattern).

## When this file is useful

This wisdom is calibrated for **one specific microarchitecture**. The
chosen factorizations and codelet variants reflect that CPU's L1d
size, port topology, FMA latency, prefetcher behavior, and memory
bandwidth.

| Your CPU | Use this wisdom? | Notes |
|----------|:---------------:|-------|
| Intel i9-14900K / 14900KF / 14900KS | yes | direct match |
| Intel 13th-gen Raptor Lake (i9-13900*, i7-13700K) | mostly yes | same uarch family, ~5% suboptimal in spots |
| Intel 12th-gen Alder Lake (i9-12900K, i7-12700K) | yes-ish | same P-core uarch (Golden Cove), slightly different cache hierarchy |
| Intel Sapphire Rapids / Emerald Rapids server | partially | server uarch has AVX-512 + larger L2 — recalibrate for accurate plans |
| AMD Zen 4 / Zen 5 | **no** | different uarch entirely; CPE numbers differ; recalibrate |
| ARM (M1/M2/M3, Graviton) | **no** | different ISA; AVX2 codelets don't apply |

If you're not on a Raptor-Lake-family Intel CPU, this file will still
load (the format is portable), but the chosen plans may be suboptimal
by 10–30%. Recalibrating on your host gives you the headline numbers.

## When to regenerate

- **Different host** — different uarch needs different wisdom.
- **After codelet changes** — running the codelet generators
  (`tools/radix_profile/extract.py` then `measure_cpe.exe`) emits new
  CPE numbers. The cost model (which feeds calibration) changes,
  potentially picking different factorizations.
- **Different bench grid** — the v1.0 wisdom covers 207 cells matching
  `build_tuned/benches/bench_1d_vs_mkl.c`. If your workload uses
  different (N, K), calibrate those cells:
  `python build_tuned/calibrate.py --cells "N1:K1,N2:K2,..."`

## How to regenerate

Quick (recommended for users who just want production speed on their
own CPU):

```
python build_tuned/calibrate.py
```

Writes `build_tuned/vfft_wisdom_tuned.txt`. Then either:
- Load from there directly: `vfft_load_wisdom("build_tuned/vfft_wisdom_tuned.txt")`
- Or copy to your project's location of choice

Runtime ~30–60 minutes on the i9-14900KF for the full 207-cell grid.
Slower on weaker hardware. Faster (~5 min) if you scope down via
`--cells`.

For a deeper "recalibrate everything from scratch" workflow including
the per-radix CPE table, see [`tools/radix_profile/README.md`](../../tools/radix_profile/README.md).

## Calibration host details

| Field | Value |
|-------|-------|
| CPU | Intel Core i9-14900KF |
| uarch | Raptor Lake (P-core: Raptor Cove) |
| ISA | AVX2 + FMA3 (no AVX-512) |
| Pinned core | logical CPU 0 (P-core 0) |
| Power plan | High Performance |
| Background load | minimal (idle) |
| Compiler | Intel ICX 2025.3 |
| Bench harness | `build_tuned/build.py --vfft --src build_tuned/benches/bench_1d_vs_mkl.c` |
| Wisdom format | v5 (per-stage variant codes: FLAT / LOG3 / T1S / BUF) |
| Cells | 198 calibrated entries across the 207-cell bench grid |

## Provenance

`wisdom.txt` here is a snapshot of `build_tuned/vfft_wisdom_tuned.txt`
at v1.0 release. The build_tuned/ copy is the live calibration target
for development; this packaged copy is what ships with the library.
