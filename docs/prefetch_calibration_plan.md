# Prefetch Calibration — Implementation Plan

Status: draft, MVP scope.
Related: [prefetch_calibration_options.md](prefetch_calibration_options.md) — the design exploration that led here.

## Goal

Make per-codelet software prefetch parameters tunable per CPU at install time,
with zero execution overhead. Prune losing variants after calibration; ship
only the winner per codelet.

Proven baseline: R=16 AVX2 t1_dit K=256 got **2.7x** from hand-tuned prefetch
on Raptor Lake. Same pattern expected for R=32 and composite radixes later.

## Model

ATLAS-style install-time tuning. Entry point: `generate_all.bat`.

- Generators are the single source of truth
- Codelets committed to git are the Raptor Lake-tuned defaults
- Running `generate_all.bat` on another CPU overwrites them with that CPU's
  winners — local diff, user commits to their own branch if they want

## Scope (MVP)

Calibrate:
- R=16 AVX2 `t1_dit_fwd`
- R=16 AVX2 `t1_dit_bwd`
- R=32 AVX2 `t1_dit_fwd`
- R=32 AVX2 `t1_dit_bwd`

Skip (proven not to benefit or out of scope for v1):
- R=2, R=4, R=8, small primes
- AVX-512 variants (no hardware to test)
- Composite radixes (R=20, R=25, ...) — add later with same mechanism

## Prefetch parameters

- `depth` ∈ {1, 2} — columns ahead to prefetch
- `count` ∈ {0, 1, 2, 3, 4, 5, 6, 7} — twiddle cache lines per slot, bounded
  by slots available per radix
- `hint` — fixed to `_MM_HINT_T0` in v1 (add T1/NTA later if needed)

## Calibration algorithm

Monotone walk with early stop:

```
best_time = measure(depth=1, count=0)   # baseline
winner    = (1, 0)

for depth in [1, 2]:
    for count in 1..max_count:
        t = measure(depth, count)
        if t < best_time:
            best_time = t
            winner = (depth, count)
        elif t > best_time * 1.02:
            break   # regression, stop this depth
```

Expected measurements per codelet: 5–10. Per codelet runtime: <1 second.

## Generator changes

CLI flags added to `gen_radix16.py` and `gen_radix32.py`:

```
--prefetch-depth N       default: Raptor Lake winner (1 for R=16)
--prefetch-count N       default: Raptor Lake winner (3 for R=16)
--prefetch-hint {T0}     default: T0 (only value supported in v1)
```

Output function name stays the same (e.g., `radix16_t1_dit_fwd_avx2`). In-place
overwrite. The prefetch config is encoded as a comment in the generated file
header:

```c
/**
 * @file fft_radix16_avx2_ct_t1_dit.h
 * @brief DFT-16 AVX2 t1 DIT
 * Generated by gen_radix16.py
 * Prefetch config: depth=1 count=3 hint=T0
 */
```

Calibration phase uses a **temporary** variant-encoded filename:
`build/generated/calibration/fft_radix16_..._pf_d1c3t0.h`. Each calibration
variant has its function name suffixed similarly so the bench can link and
call them directly. These files are deleted after the bench picks a winner.

## Calibration bench

`bench/bench_pf_calibrate.c`:

- Includes all calibration variant headers from `build/generated/calibration/`
- For each calibrated codelet, iterates the (depth, count) grid with monotone
  walk + early stop
- At each config, runs the variant for fixed duration (e.g., 500 ms) to stable ns/call
- Writes winner per codelet to stdout + `build/generated/calibration_report.txt`
- Emits the winner in a form the calling script can parse

Output format (stdout, one line per codelet):
```
WINNER R16_t1_dit_fwd depth=1 count=3 hint=T0 ns=1631.2
WINNER R16_t1_dit_bwd depth=1 count=3 hint=T0 ns=1628.5
WINNER R32_t1_dit_fwd depth=1 count=4 hint=T0 ns=3142.0
WINNER R32_t1_dit_bwd depth=1 count=4 hint=T0 ns=3138.4
```

## generate_all.bat extension

```bat
REM ... existing generator invocations ...

REM === Prefetch calibration ===
echo Generating prefetch calibration variants...
for %%D in (1 2) do (
    for %%C in (0 1 2 3 4 5 6 7) do (
        python src/stride-fft/generators/gen_radix16.py ^
            --variant ct_t1_dit ^
            --prefetch-depth %%D --prefetch-count %%C --prefetch-hint T0 ^
            --output build/generated/calibration/fft_radix16_..._pf_d%%Dc%%Ct0.h
        REM ... same for gen_radix32 ...
    )
)

echo Compiling calibration bench...
cmake --build build --target vfft_bench_pf_calibrate

echo Running calibration...
build/bin/vfft_bench_pf_calibrate.exe > build/generated/calibration_report.txt

echo Regenerating winners in-place...
REM Parse calibration_report.txt, extract winner configs
REM Invoke generators with winner configs, overwrite committed codelets
python src/stride-fft/generators/apply_winners.py ^
    --report build/generated/calibration_report.txt

echo Cleaning calibration artifacts...
rmdir /s /q build/generated/calibration

echo Done. Check git diff for tuned codelet changes.
```

## Helper script: apply_winners.py

Parses `calibration_report.txt`, invokes the generators with winner configs,
overwrites `src/stride-fft/codelets/avx2/*.h` in place. Small script, 30-50
lines of Python.

## CMake changes

Minimal. One new target:

```cmake
add_executable(vfft_bench_pf_calibrate
    src/stride-fft/bench/bench_pf_calibrate.c
)
target_include_directories(vfft_bench_pf_calibrate PRIVATE
    ${CMAKE_SOURCE_DIR}/src/stride-fft
    ${CMAKE_BINARY_DIR}/generated/calibration
)
```

Build target only when calibration artifacts exist. Otherwise skipped.

## Test + verify

1. Run `generate_all.bat` on Raptor Lake → expect:
   - `fft_radix16_avx2_ct_t1_dit.h` regenerated with `depth=1 count=3`
   - `fft_radix32_avx2_ct_t1_dit.h` regenerated with winner from calibration
   - No regression in `vfft_bench_codelet 256 3000 16` or `vfft_bench_codelet 256 3000 32`
2. Manual smoke test: force `count=0` in generator, verify R=16 perf drops
   back to ~4474 ns (the baseline we measured pre-prefetch).
3. Full FFT test: `vfft_bench N K` at sizes where R=16 and R=32 factor in,
   confirm no regression.

## Open loose ends (for later)

- Ship Raptor Lake winner as committed default — confirmed yes, but the bwd
  variant needs explicit calibration to confirm `d1c3t0` is also its winner
  (currently assumed, not measured separately).
- Composite radixes (R=20, R=25) — add to calibration list when those
  generators are touched. No architectural change needed.
- AVX-512 variants — skip until hardware available.
- `hint` axis (T1/NTA) — add later if profiling shows L1 contention that T0
  can't handle.
- `gitignore` strategy for calibrated codelets — noisy local diffs accepted
  for now; revisit when the idea has proven itself.
